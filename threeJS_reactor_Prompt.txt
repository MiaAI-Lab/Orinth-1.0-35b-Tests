Create a complete, production-quality Three.js scene in a single self-contained HTML file.

Goal:
Build an interactive 3D “futuristic energy reactor” scene that demonstrates strong Three.js skills, visual design, animation, lighting, materials, and performance awareness.

Requirements:
1. Use Three.js only from a CDN.
2. The output must be a single HTML file with embedded CSS and JavaScript.
3. Do not use external assets, images, textures, models, or paid libraries.
4. The scene must include:
   - A central animated reactor core.
   - Multiple orbiting energy rings or particles.
   - At least 3 different geometric objects.
   - Real-time animation using requestAnimationFrame.
   - Responsive resizing.
   - Camera controls using OrbitControls.
   - Dynamic lighting, including at least one moving light.
   - Custom materials using emissive, metalness, roughness, transparency, or shaders.
   - A visually pleasing background, floor/grid, or environment.
5. Add user interaction:
   - Mouse movement or pointer interaction should subtly affect the scene.
   - Clicking the reactor should trigger a visible animation or color change.
6. Add a minimal UI overlay showing:
   - Scene title.
   - Short instructions.
   - Current animation state or reactor power level.
7. Optimize for smooth performance:
   - Avoid excessive geometry.
   - Reuse materials where sensible.
   - Use efficient animation loops.
8. Code quality matters:
   - Clean structure.
   - Meaningful variable names.
   - Helpful comments.
   - No broken imports.
   - No missing dependencies.
9. The final answer must contain only the full HTML code, with no explanation before or after.

Judging criteria:
- Runs correctly on first paste into an HTML file.
- Visual quality and creativity.
- Correct use of Three.js fundamentals.
- Smooth animation.
- Interactivity.
- Code organization.
- Responsiveness.
